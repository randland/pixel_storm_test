# Context Modules Index

> **Load this first** to determine which context module you need.

## Module Selection

| Module | Load When |
|--------|-----------|
| `learning-phases.md` | Understanding curriculum structure, phase definitions |
| `platform-specs.md` | Technical setup, architecture, installation |
| `progress-tracking.md` | Spaced repetition, skill assessments, session notes |

## Decision Tree

```
Need to know what phase we're in or what comes next?
  └─> learning-phases.md

Need technical specs, dependencies, or architecture?
  └─> platform-specs.md

Need to track skills, review concepts, or log sessions?
  └─> progress-tracking.md
```

## Module Summaries

### learning-phases.md
**Contains**:
- Phase 1-4 curriculum breakdown (TresJS → WebGPU → TSL → Creative)
- Checkbox items for each learning objective
- Background skills assessment
- Learning approach description (Three.js examples style)

**Use for**: Planning what to learn next, understanding phase structure

### platform-specs.md
**Contains**:
- Technology stack details (Vue 3, TresJS, Three.js, WebGPU)
- System requirements and prerequisites
- Installation commands
- Project structure
- Configuration (WebGPU browser setup, TresJS config)
- Development workflow and troubleshooting

**Use for**: Setting up environment, understanding architecture, debugging setup issues

### progress-tracking.md
**Contains**:
- Technical proficiency progress bars
- Platform development progress
- Spaced repetition tracking table (concept, last review, confidence)
- Skill mastery indicators
- Session notes template

**Use for**: Recording progress, scheduling reviews, tracking which concepts need reinforcement
